Frontend:

1- SPA
Decidi fazer o site SPA, devido sua natureza (Ser um site com poucas views, menor)
Isso é importante por basicamente me obrigar a fazer um routing "manual"
Seria possível fazer um routing de barras ao invés do padrão de hashs de um browser, mas levaria bastante tempo para desenvolver um.

2- Integração com PHP

Como o site precisa ter um db com interface em php, somado com o fato do site já ter sua estrutura baseada em vanilla JS, vou tratar o php
como uma API/backend e dar fetch nele

3- Organização de pastas

Modularizei o JavaScript para dividir as funções
Separei uma folder para a API em PhP, os arquivos de estilização e lógica do frontend ficam no assets/
O banco de dados tem sua própria pasta

4- Databasae

Foi feita uma transição do productsData.JS para o db, para isso eu fiz um seed com produtos iniciais

Backend

É dividido nas pastas

/api/      <- para os arquivos php que fazem o crud
/database/ <- para o banco de dados (schema e seed)

db.php faz a conexão, todos os outros php usam ele para conectar com o db
o backend aqui funciona como uma API, logo é um require_once que serve como listener
